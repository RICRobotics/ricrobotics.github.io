GitHub, while a similar product conceptually, is a completely different service - to use it, you'll need to make an account at: github.com

git and GitHub are respectively some of the most popular development tools in the world, and as such there are no shortage of resources to familiarize yourself with them on your own. Git on Wikipedia and GitHub on Wikipedia are probably reasonable places to start.


**The Basic Idea of the Two Services**

git is a sort of file manager, designed to make the process of sharing code and editing the same codebase at the same time as someone else as seamless as possible. The idea is: if you have a repo which you are editing with your friend, you might use a harddrive to sync them such that they are in the same state. Then, you can each go home and work on your own parts of the program. BUT: what if you both edit the same file, say, one of you edits functionA and the other functionB? How would you combine the file when you meet up again? Git handles this and much more, so you almost don't need to worry about those problems. At the same time it keeps a detailed history of commits you made.

GitHub is built on git - it largely uses the git software (which is open source), all it does, in theory, is offer cloud computing, so that instead of needing to meet up with your friend to sort out file changes, or setting up a peer to peer connection, one of you can push to GitHub's servers, and then at a later time, the other can pull.
